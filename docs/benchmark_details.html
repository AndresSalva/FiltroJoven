Guía Detallada del Benchmark GA

Cómo se ejecuta, qué produce y cómo interpretar sus resultados

1. Propósito general

El script experiments/benchmark.py automatiza la evaluación del algoritmo genético que impulsa el filtro de rejuvenecimiento facial. Permite responder de manera objetiva cuáles combinaciones de operadores ofrecen mejor rendimiento, qué tan estables son y cómo evoluciona la población a lo largo de las generaciones.

2. Flujo operativo

  1. Carga y preparación. Se abre la imagen objetivo y se calculan máscaras faciales para aplicar la transformación de manera localizada.
  2. Definición de configuraciones. Se combinan todas las funciones de aptitud, operadores de selección y cruces indicados en la CLI. En el barrido completo se evaluaron 48 configuraciones.
  3. Ejecución multihilo. Cada combinación se repite --runs veces (10 en el estudio final) con semillas distintas derivadas del base_seed.
  4. Registro de resultados. Se almacenan aptitudes, parámetros, historiales por generación, diversidad poblacional y duración de cada corrida; opcionalmente se guardan las imágenes finales.
  5. Generación de salidas. El benchmark produce CSV, figuras comparativas y un reporte Markdown listo para revisión rápida.

3. Métricas y archivos de salida

3.1 Datos (CSV)

  • ga_benchmark_runs.csv: detalle por corrida (aptitud, parámetros, historial, diversidad y duración).
  • ga_benchmark_summary.csv: agregados por combinación, con medias y desviaciones de cada métrica, además de valores normalizados (mean_cdf, mean_z).
  • ga_benchmark_history.csv: evolución por generación (mejor aptitud, mejor histórico y diversidad).

3.2 Figuras (PNG)

Archivo Qué muestra Cómo interpretarlo
fitness_boxplot_cdf.png Distribución de la mejor aptitud normalizada por fitness y selección. Destaca configuraciones con percentiles altos y baja varianza.
heatmap_*_cdf.png Promedio de CDF por selección × cruce para cada fitness. Los colores más intensos indican combinaciones superiores.
convergence_*_cdf.png Curvas de convergencia normalizadas por generación (con ±1 std). Permiten ver rapidez y estabilidad del progreso.
diversity_* .png Diversidad (desviación típica media de parámetros) en el tiempo. Ayudan a detectar pérdida de diversidad o exploración excesiva.

3.3 Reporte autogenerado

El archivo benchmark_results_assignment/reports/benchmark_report.md resume configuración, resultados normalizados y lista de figuras. Es ideal para una verificación rápida tras la ejecución.

4. Normalización y comparación

Las funciones de aptitud original, ideal y color operan en escalas distintas. Para compararlas, el benchmark calibra cada fitness con muestras aleatorias (media y desviación) y transforma cada resultado a Z-score y luego a CDF (0-1). Así, un valor de 0.99 representa el mismo percentil sin importar el fitness evaluado.

5. Interpretación de resultados

  1. Rendimiento promedio: revisar ga_benchmark_summary.csv y el boxplot para identificar combinaciones con CDF alto y baja desviación.
  2. Análisis por operador: los heatmaps muestran la intensidad de cada selección × cruce para cada fitness.
  3. Velocidad de convergencia: las curvas de convergencia evidencian qué configuraciones escalan rápido hacia percentiles altos.
  4. Diversidad: monitorizar las gráficas de diversidad para evitar estancamientos o poblaciones excesivamente dispersas.
  5. Selección de imágenes: ejecutar con --save-images para evaluar la calidad visual de las mejores corridas.

6. Recomendaciones prácticas

  • Exploración rápida: utilizar valores bajos de --runs y --gens sin guardar imágenes, solo para validar dependencias.
  • Barrido formal: mantener 10 corridas y 20 generaciones, activando --save-images para obtener evidencia visual completa.
  • Selección final: usar el resumen y las figuras para elegir la mejor configuración, confirmando con las imágenes asociadas.
  • Documentación: complementar esta guía con el informe de asignación y el reporte autogenerado.
