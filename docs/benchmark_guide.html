Guia Practica del Benchmark

Resumen operativo de experiments/benchmark.py

1. Objetivo rapido

El benchmark genera y evalua todas las combinaciones de funciones de aptitud, selecciones y cruces que se indiquen. Produce datos cuantitativos y material visual para entender el comportamiento real del algoritmo genetico y documentar los hallazgos sin esfuerzo manual.

2. Ejecucion basica

  1. Instala dependencias: pip install -r requirements.txt.
  2. Coloca la imagen de prueba en input_images/.
  3. Corrida rapida de validacion: 
    python -u experiments/benchmark.py --image tuto.jpg --runs 1 --gens 2 --pop 6 `
    --fitness original ideal color --selection tournament --crossover single_point `
    --mutation gaussian --output-dir benchmark_results_quick
  4. Barrido completo recomendado: 
    python -u experiments/benchmark.py --image tuto.jpg --runs 10 --gens 20 --pop 24 `
    --fitness original ideal color `
    --selection tournament roulette rank sus `
    --crossover single_point two_point k_point uniform `
    --mutation gaussian --calibration-samples 40 --save-images `
    --output-dir benchmark_results

Reduce corridas o generaciones durante la iteracion. Activa --save-images solo cuando necesites los resultados finales.

3. Carpetas y archivos

Ruta Contenido y uso
benchmark_results_*/data/ CSV con corridas individuales (ga_benchmark_runs.csv), resumenes (ga_benchmark_summary.csv) e historiales (ga_benchmark_history.csv).
benchmark_results_*/figures/ Boxplots, heatmaps, curvas de convergencia y diversidad en escala CDF 0-1.
benchmark_results_*/reports/ benchmark_report.md: configuracion, top combinaciones y listado de figuras.
benchmark_results_*/images/<fitness>/ Imagen final de cada corrida (solo si se usa --save-images). Ignorada en Git para evitar peso.

4. Interpretacion rapida

  • Promedio de rendimiento: ordena ga_benchmark_summary.csv por mean_cdf; valores cercanos a 1 indican percentiles altos.
  • Estabilidad: revisa std_fitness y std_cdf. Valores bajos implican corridas consistentes.
  • Convergencia: las figuras convergence_*_cdf.png muestran cuan rapido progresa cada configuracion.
  • Diversidad: diversity_* permite detectar estancamientos o exceso de exploracion.
  • Revision visual: usa las imagenes finales para comparar resultados cualitativos.

5. Sugerencias finales

  • Realiza corridas cortas sin --save-images hasta confirmar dependencias y rutas antes del barrido definitivo.
  • Complementa esta guia con docs/benchmark_assignment_report.md y docs/benchmark_details.md para un informe completo.
  • Si agregas nuevos operadores o funciones de aptitud, actualiza los registros en experiments/benchmark.py y repite el benchmark.
  • Evita versionar las imagenes generadas; el .gitignore ya excluye benchmark_results_*/images/.
