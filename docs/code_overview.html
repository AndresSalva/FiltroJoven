Mapa de modulos y recomendaciones para extender el proyecto

1. Piezas principales

Archivo Rol
main.py Punto de entrada para una sola corrida; expone CLI para imagen, fitness y operadores.
transformations/face_manipulator.py Mascaras faciales, transformacion parametrica y motor del algoritmo genetico.
ga/fitness_functions.py Funciones original, ideal, color y combinaciones ponderadas.
ga/selection_operators.py Torneo, ruleta, rank y SUS.
ga/crossover_operators.py Cruces single-point, two-point, k-point y uniform.
ga/mutation_operators.py Mutacion gaussiana y recorte de parametros via clip_params.
experiments/benchmark.py Barrido masivo, generacion de CSV, figuras y reportes.
utils/ Funciones de imagen, metricas y visualizacion.
core/ Deteccion facial y mascaras base (Haar cascades, landmarks).

2. Flujo tipico (main.py)

  1. Parseo de argumentos (imagen, fitness, operadores, generaciones, poblacion, semilla).
  2. Carga de la imagen con utils.image_utils.load_image.
  3. Mapeo de nombres a funciones reales (FITNESS_NAMES, SELECTION_MAP, etc.).
  4. Ejecucion de run_genetic_algorithm con mascaras precalculadas.
  5. Guardado de la mejor imagen en output_images/ y log de parametros en consola.

3. Transformacion y GA (transformations/face_manipulator.py)

  • PARAM_BOUNDS: limites para bilateral_d, sigma_color, sigma_space, gamma, unsharp_amount.
  • sample_params: muestreo de parametros dentro de los limites (deterministico si se provee generador).
  • apply_transformation: aplica suavizado bilateral, correccion gamma y unsharp masking selectivo.
  • run_genetic_algorithm: inicializa poblacion, aplica seleccion, cruce, mutacion y elitismo; puede devolver historial completo para analisis.

4. Fitness y operadores (ga/)

  • Fitness: `fitness_functions.py` define las funciones base y herramientas para combinarlas con pesos.
  • Seleccion: `selection_operators.py` implementa torneo, ruleta, rank y SUS.
  • Cruce: `crossover_operators.py` mezcla diccionarios de parametros (single, two, k, uniform).
  • Mutacion: `mutation_operators.py` aplica ruido gaussiano proporcional al rango de cada parametro y recorta valores.

5. Benchmark (experiments/benchmark.py)

  1. Genera combinaciones segun la CLI y ejecuta el GA con semillas unicas.
  2. Guarda imagenes finales opcionalmente y registra cada corrida.
  3. Construye ga_benchmark_runs.csv, ga_benchmark_summary.csv y ga_benchmark_history.csv.
  4. Produce boxplots, heatmaps y curvas de convergencia/diversidad.
  5. Genera un reporte autogenerado con configuracion y rutas relevantes.

6. Extensiones recomendadas

  • Nuevas funciones de aptitud: añadelas a `fitness_functions.py`, registra el nombre en los mapas y actualiza la CLI si deseas incluirlas en el benchmark.
  • Operadores personalizados: agrega la funcion en el modulo correspondiente y enlazala en los diccionarios utilizados por `main.py` y el benchmark.
  • Transformaciones adicionales: ajusta `PARAM_BOUNDS` y `apply_transformation`, cuidando la conversion final a `uint8`.
  • Depuracion rapida: usa pocas generaciones/poblacion o el benchmark con `--runs 1 --gens 2` para reproducir problemas sin esperar el barrido completo.
